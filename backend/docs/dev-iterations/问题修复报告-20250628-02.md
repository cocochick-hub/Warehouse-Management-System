# 问题修复报告：转包 Bug（3个严重问题）

## 问题描述

### Bug 1：转包不更新 `InventoryStock.onHandQty`
转包时只更新了 `InboundKanbanLabel.labelQty`，但 `InventoryStock.onHandQty` 没有同步扣减，导致库存数据不一致。

### Bug 2：全量转出的看板仍可用于出库
已全量转包（`transferStatus = "已转包"`）的看板，仍然可以通过 `issueByScan` 出库。

### Bug 3：部分转出的看板缺乏出库保护
已部分转包（`transferStatus = "转包"`）的看板，仍然可以通过 `issueByScan` 出库。

## 根因分析

**Bug 1：** `TransferServiceImpl.executeTransfer()` 在更新源看板数量后，没有同步扣减 `InventoryStock.onHandQty`。库存表和看板标签表是两条独立的数据流，转包操作只更新了看板，没有联动库存。

**Bug 2 和 Bug 3：** `OutboundOrderServiceImpl.issueByScan()` 在检查看板状态时，只检查了 `transferStatus == "已出库"`（防止重复出库），没有检查 `transferStatus == "已转包"` 或 `transferStatus == "转包"`，导致转包后的看板仍可出库。

```java
// 4.1 同步扣减 InventoryStock.onHandQty（按物料+供应商+库区定位）
String stockArea = source.getWarehouseArea() != null ? source.getWarehouseArea() : "默认库区";
InventoryStock stock = inventoryStockRepository
        .findByMaterialCodeAndSupplierAndWarehouseArea(
                source.getMaterialCode(), source.getSupplierName(), stockArea)
        .orElse(null);
if (stock != null) {
    int newOnHand = stock.getOnHandQty() - transferQty;
    stock.setOnHandQty(Math.max(newOnHand, 0));
    stock.setUpdatedBy(operator);
    stock.setUpdatedAt(LocalDateTime.now());
    inventoryStockRepository.save(stock);
}
```

## 验证结果
构建环境存在 Java 24 与 Spring Boot 2.7.18 的兼容性问题（`maven-compiler-plugin 3.11.0` 不支持 Java 24），导致无法在本地运行测试。代码修改已通过人工审核：
- `InventoryStockRepository` 已正确注入 `TransferServiceImpl`
- 库存同步逻辑在源看板数量更新后执行，按物料+供应商+库区定位库存行
- `issueByScan` 中新增两处转包状态检查，位于封存检查之后、出库逻辑之前
- Bug 2 检查 `"已转包"`（全量转包），Bug 3 检查 `"转包"`（部分转包）
