# 问题修复报告：AdjustDetail FK 约束错误

## 问题描述
盘点盈亏调整时，`OutboundHistory` 的 `outbound_order_id` 和 `outbound_detail_id` 被设为 `0L`，但这两个字段的外键约束（参照 `outbound_order` 和 `outbound_order_detail` 表）要求记录存在，导致插入历史记录失败或数据不一致。

盘点产生的入库/出库是虚拟操作，并没有对应的真实出库单，因此这两个字段应该为 `null`。

## 根因分析
`CheckServiceImpl.adjustDetail()` 方法在处理盘盈（diff > 0）和盘亏（diff < 0）时，错误地将 `outbound_order_id` 和 `outbound_detail_id` 设为 `0L`。但 schema 中这两个字段已改为 `DEFAULT NULL`，允许空值，盘点虚拟记录应当使用 null 而非 0。

## 代码 diff

**文件：** `src/main/java/com/example/wms/service/impl/CheckServiceImpl.java`

### 盘盈时（第 166-167 行）
```diff
- inbound.setOutboundOrderId(0L);
- inbound.setOutboundDetailId(0L);
+ inbound.setOutboundOrderId(null);
+ inbound.setOutboundDetailId(null);
```

### 盘亏时（第 182-183 行）
```diff
- outbound.setOutboundOrderId(0L);
- outbound.setOutboundDetailId(0L);
+ outbound.setOutboundOrderId(null);
+ outbound.setOutboundDetailId(null);
```

## 验证结果
构建环境存在 Java 24 与 Spring Boot 2.7.18 的兼容性问题（`maven-compiler-plugin 3.11.0` 不支持 Java 24），导致无法在本地运行测试。代码修改已通过人工审核：
- 两处 `0L` 均已替换为 `null`
- 字段类型为 `Long`，允许 null 值
- schema 中 `outbound_order_id` 和 `outbound_detail_id` 定义为 `BIGINT DEFAULT NULL`，外键约束也允许 NULL
