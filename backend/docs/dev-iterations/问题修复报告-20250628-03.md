# 问题修复报告：Schema 验证

## 问题描述
确认 `outbound_history` 表的 `outbound_order_id` 和 `outbound_detail_id` 字段已正确修改为 `DEFAULT NULL`，允许盘点虚拟记录不关联真实出库单。

## 根因分析
前期 `outbound_history` 表的这两个字段定义为 `NOT NULL`，导致盘点产生的虚拟出库/入库历史无法插入（盘点没有真实出库单）。Schema 已预先修改为 `DEFAULT NULL`，本次验证确认修改正确。

## 代码 diff

**文件：** `src/main/resources/schema.sql`（生产 schema）

```sql
-- outbound_history 表定义（第 329-351 行）
CREATE TABLE IF NOT EXISTS outbound_history (
    id                  BIGINT AUTO_INCREMENT PRIMARY KEY,
    outbound_order_id   BIGINT       DEFAULT NULL,   -- 原: NOT NULL
    outbound_detail_id  BIGINT       DEFAULT NULL,   -- 原: NOT NULL
    doc_no              VARCHAR(50)  NOT NULL,
    material_code       VARCHAR(50)  NOT NULL,
    ...
    FOREIGN KEY (outbound_order_id) REFERENCES outbound_order (id),
    FOREIGN KEY (outbound_detail_id) REFERENCES outbound_order_detail (id)
);
```

**文件：** `src/test/resources/schema-h2.sql`（测试 schema）

```sql
-- outbound_history 表定义（第 196-212 行）
CREATE TABLE IF NOT EXISTS outbound_history (
    id                  BIGINT AUTO_INCREMENT PRIMARY KEY,
    outbound_order_id   BIGINT       DEFAULT NULL,   -- 原: NOT NULL
    outbound_detail_id  BIGINT       DEFAULT NULL,   -- 原: NOT NULL
    doc_no              VARCHAR(50)  NOT NULL,
    material_code       VARCHAR(50)  NOT NULL,
    ...
    FOREIGN KEY (outbound_order_id) REFERENCES outbound_order (id)
);
```

## 验证结果
两个 schema 文件中，`outbound_order_id` 和 `outbound_detail_id` 均已定义为 `BIGINT DEFAULT NULL`，外键约束保留但允许 NULL 值。配合 CheckServiceImpl 中将 `0L` 改为 `null` 的修改，盘点产生的虚拟记录可以正确插入。
