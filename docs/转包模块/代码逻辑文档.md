# 转包模块代码逻辑文档

## 1. 核心代码解析

### 1.1 后端核心服务 (TransferServiceImpl)

#### 1.1.1 executeTransfer 方法概述

`executeTransfer` 方法是转包功能的核心，包含完整的转包流程处理。

**转包模式判断逻辑：**

```java
// ==================== 3. 判断转包模式：拆包 OR 合包 ====================
String targetKanbanNo = request.getTargetKanbanNo();
boolean userSpecifiedTarget = targetKanbanNo != null && !targetKanbanNo.trim().isEmpty();
boolean isMergeMode = false;

InboundKanbanLabel targetLabel = null;

if (userSpecifiedTarget) {
    // 用户指定了目标看板号：查找是否已存在
    targetLabel = kanbanLabelRepository.findByKanbanNo(targetKanbanNo).orElse(null);
    if (targetLabel != null) {
        // 目标看板已存在 → 合包模式
        isMergeMode = true;
        // 合包校验：物料一致性、状态、封存
    }
    // 如果目标看板不存在，targetKanbanNo保持用户指定的值，用于创建新看板
} else {
    // 用户未指定 → 自动生成目标看板号
    targetKanbanNo = generateTargetKanbanNo(source.getKanbanNo());
}
```

**两种模式的处理差异：**

| 步骤 | 拆包（isMergeMode=false） | 合包（isMergeMode=true） |
|------|---------------------------|-------------------------|
| 目标看板 | 创建新的 `InboundKanbanLabel` | 更新已有看板的 `labelQty` |
| 目标入库单 | 创建新的 `InboundOrder` | 复用原目标看板的入库单 |
| 目标库存 | `onHandQty += transferQty` | `onHandQty += transferQty` |
| 出库单类型 | "拆包出库" | "合包出库" |

#### 1.1.2 创建源出库单（两种模式相同）

```java
// ========== 4. 创建源看板的出库单（两种模式都需要）====================
String outboundDocNo = generateOutboundDocNo();
OutboundOrder outboundOrder = new OutboundOrder();
outboundOrder.setDocNo(outboundDocNo);
outboundOrder.setSupplier(source.getSupplierName());
outboundOrder.setStatus("已完成");
outboundOrder.setItemCount(1);
outboundOrder.setPlannedTotalQty(transferQty);
outboundOrder.setActualTotalQty(transferQty);
outboundOrder.setOutboundType(isMergeMode ? "合包出库" : "拆包出库");
outboundOrder.setRemark(isMergeMode
        ? "合包出库，源看板: " + source.getKanbanNo() + " → 目标看板: " + targetKanbanNo
        : "拆包出库，源看板: " + source.getKanbanNo() + " → 新目标看板: " + targetKanbanNo);
outboundOrderRepository.save(outboundOrder);

OutboundOrderDetail outboundDetail = new OutboundOrderDetail();
outboundDetail.setOutboundOrderId(outboundOrder.getId());
outboundDetail.setDocNo(outboundDocNo);
// ... 设置其他字段 ...
outboundOrderDetailRepository.save(outboundDetail);
```

**解析：**
- 生成唯一的出库单号（格式：`OUT` + 时间戳 + 序号）
- 出库类型标记为"转包出库"，便于区分
- 备注中记录源看板号，便于追溯

#### 1.1.3 扣减源库存

```java
// ========== 步骤2：扣减源物料库存 ==========
String stockArea = source.getWarehouseArea() != null ? source.getWarehouseArea() : "默认库区";
InventoryStock sourceStock = inventoryStockRepository
        .findByMaterialCodeAndSupplierAndWarehouseArea(
                source.getMaterialCode(), source.getSupplierName(), stockArea)
        .orElse(null);
if (sourceStock != null) {
    int newOnHand = sourceStock.getOnHandQty() - transferQty;
    sourceStock.setOnHandQty(Math.max(newOnHand, 0));  // 防止负数
    inventoryStockRepository.save(sourceStock);
}
```

**解析：**
- 按"物料+供应商+库区"定位库存
- 使用 `Math.max(newOnHand, 0)` 防止库存变为负数
- 与出库单记录保持一致

#### 1.1.4 处理目标看板（拆包/合包分支）

```java
// ==================== 6. 处理目标看板（拆包/合包不同处理）====================
if (isMergeMode) {
    // ========== 合包模式 ==========
    // 6a. 更新目标看板数量
    int targetQtyBefore = targetLabel.getLabelQty();
    targetLabel.setLabelQty(targetQtyBefore + transferQty);
    targetLabel.setTransferStatus("转入");
    kanbanLabelRepository.save(targetLabel);

    // 6b. 增加目标物料库存（与源同库区）
    InventoryStock targetStock = inventoryStockRepository
            .findByMaterialCodeAndSupplierAndWarehouseArea(
                    source.getMaterialCode(), source.getSupplierName(), stockArea)
            .orElse(null);
    if (targetStock != null) {
        targetStock.setOnHandQty(targetStock.getOnHandQty() + transferQty);
        inventoryStockRepository.save(targetStock);
    }
    // 合包时使用目标看板关联的原入库单号
    inboundDocNo = targetLabel.getDocNo();

} else {
    // ========== 拆包模式 ==========
    // 6a. 创建新的入库单
    inboundDocNo = generateInboundDocNo();
    InboundOrder inboundOrder = new InboundOrder();
    inboundOrder.setDocNo(inboundDocNo);
    // ... 设置其他字段 ...
    inboundOrderRepository.save(inboundOrder);

    // 6b. 创建入库明细
    InboundOrderDetail inboundDetail = new InboundOrderDetail();
    inboundDetail.setInboundOrderId(inboundOrder.getId());
    // ... 设置其他字段 ...
    inboundOrderDetailRepository.save(inboundDetail);

    // 6c. 增加目标物料库存（同上）
    // 6d. 创建目标看板标签
    InboundKanbanLabel target = new InboundKanbanLabel();
    target.setKanbanNo(targetKanbanNo);
    target.setLabelQty(transferQty);
    // ...
    kanbanLabelRepository.save(target);
}
```

**合包模式解析：**
- 直接更新目标看板的 `labelQty`，累加转移数量
- 不创建新的入库单，复用目标看板关联的原入库单
- 目标看板 `transferStatus` 设为"转入"

**拆包模式解析：**
- 创建新的入库单和入库明细
- 创建新的看板标签，关联新入库单
- 目标看板号可以是用户指定或自动生成

**解析：**
- 目标看板关联新入库单和入库明细
- 看板号由用户指定或自动生成（`T-` 前缀）
- 状态为"已入库"，转包状态为"转入"

#### 1.1.7 更新源看板状态

```java
// ========== 步骤6：更新源看板状态 ==========
int sourceQtyAfter = sourceQty - transferQty;
source.setLabelQty(sourceQtyAfter);
source.setTransferStatus(sourceQtyAfter == 0 ? "已转包" : "转包");
kanbanLabelRepository.save(source);
```

**解析：**
- 部分转包（还有剩余）：状态设为"转包"
- 全量转包（数量归零）：状态设为"已转包"

#### 1.1.8 单号生成器

```java
private synchronized String generateOutboundDocNo() {
    String timestamp = LocalDateTime.now().format(DOC_NO_FORMATTER);
    for (int attempt = 0; attempt < 1000; attempt++) {
        int suffix = DOC_NO_SEQUENCE.updateAndGet(current -> (current + 1) % 1000);
        String docNo = "OUT" + timestamp + String.format("%03d", suffix);
        if (!outboundOrderRepository.existsByDocNo(docNo)) {
            return docNo;
        }
    }
    throw new IllegalStateException("出库单号生成失败，请稍后重试");
}
```

**解析：**
- 使用 `synchronized` 保证多线程下单号唯一
- 时间戳格式：`yyyyMMddHHmmssSSS`
- 序号范围：000~999，循环使用
- 最大重试1000次，避免无限循环

#### 1.2.3 执行转包

```javascript
async function handleExecuteTransfer() {
  if (!selectedKanban.value) {
    ElMessage.warning('请先选择看板')
    return
  }
  if (transferForm.transferQty > selectedKanban.value.availableQty) {
    ElMessage.warning('转包数量不能超过可用数量')
    return
  }

  try {
    await ElMessageBox.confirm(
      `确认将 ${selectedKanban.value.kanbanNo} 转出 ${transferForm.transferQty} 件物料？`,
      '转包确认',
      { type: 'warning' }
    )
  } catch {
    return
  }

  submitting.value = true
  try {
    const { data } = await executeTransfer({
      sourceKanbanNo: selectedKanban.value.kanbanNo,
      targetKanbanNo: transferForm.targetKanbanNo || undefined,
      transferQty: transferForm.transferQty
    })
    transferResult.value = data
    resultVisible.value = true
    fetchKanbans()  // 刷新列表
  } finally {
    submitting.value = false
  }
}
```

**解析：**
- 执行前再次校验数量
- 弹出确认框防止误操作
- 成功后显示结果对话框
- 刷新看板列表

## 2. 设计方案解析

### 2.1 为什么需要创建新的入库单和出库单？

**原始需求**：将源看板的物料转移到目标看板

**实现方式对比**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| 仅更新看板数量 | 简单，改动少 | 数据不完整，缺少出入库记录 |
| 创建新的入库单+出库单 | 数据完整，可追溯 | 实现复杂 |

**最终选择**：创建完整的出入库记录

**理由**：
1. **数据完整性**：保证库存变动与单据一致
2. **可追溯性**：可以通过单据号追溯转包来源和去向
3. **审计需求**：财务和库存审计需要完整的出入库记录

### 2.2 事务保证

转包操作涉及多个表的写操作：
- `outbound_order` + `outbound_order_detail`（出库单）
- `inbound_order` + `inbound_order_detail`（入库单）
- `inventory_stock`（库存，扣减+增加）
- `inbound_kanban_label`（目标看板）
- `inbound_kanban_label`（源看板状态更新）
- `package_transfer`（转包历史）

**解决方案**：使用 `@Transactional` 注解

```java
@Override
@Transactional
public TransferResultDTO executeTransfer(TransferRequest request, String operator) {
    // ... 所有数据库操作都在一个事务中 ...
}
```

**事务边界**：
- 方法开始 → 创建事务
- 方法正常返回 → 提交事务
- 方法异常抛出 → 回滚事务

### 2.3 单号生成策略

**问题**：分布式环境下如何保证单号唯一？

**解决方案**：时间戳 + 序号 + 数据库唯一性校验

```java
private synchronized String generateOutboundDocNo() {
    String timestamp = LocalDateTime.now().format(DOC_NO_FORMATTER);
    for (int attempt = 0; attempt < 1000; attempt++) {
        int suffix = DOC_NO_SEQUENCE.updateAndGet(current -> (current + 1) % 1000);
        String docNo = "OUT" + timestamp + String.format("%03d", suffix);
        if (!outboundOrderRepository.existsByDocNo(docNo)) {
            return docNo;
        }
    }
    throw new IllegalStateException("出库单号生成失败，请稍后重试");
}
```

**解析**：
- `synchronized`：保证同一JVM实例内单号不重复
- 时间戳：区分不同时间生成的單據
- 序号：同一毫秒内可生成1000个不同单号
- 数据库校验：确保唯一性（双重保障）

### 2.4 库存定位策略

**问题**：如何确定从哪个库区扣减/增加库存？

**解决**：按源看板的库区定位

```java
String stockArea = source.getWarehouseArea() != null ? source.getWarehouseArea() : "默认库区";
```

**解析**：
- 目标看板继承源看板的库区
- 统一"物料+供应商+库区"作为库存定位键
- 如果库区不存在，则创建新的库存记录
