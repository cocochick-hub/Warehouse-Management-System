# admin 权限与转包入库单按钮修复 - 测试文档

## 测试目标

验证本次修复是否解决：

1. admin 登录后访问管理页面被误判无权限。
2. 转包页面新建入库单入口没有实际动作。
3. 转包完成后无法查看目标入库单。

## 手工测试用例

### TC-01：admin 访问用户权限管理

步骤：

1. 使用 admin 账号登录。
2. 点击侧边栏“用户权限”。
3. 进入 `/system/users`。

预期结果：

- 页面正常打开。
- 不再提示“无权限访问该页面”。

修复点：

- 路由守卫从 `getUser()` 读取 `wms_user`。

### TC-02：admin 访问基础信息页面

步骤：

1. 使用 admin 账号登录。
2. 进入基础信息下的物料、包装、供应商、库区页面。

预期结果：

- admin 可访问。
- 父级路由的 `roles: ['admin', 'manager']` 正常生效。

修复点：

- 路由守卫通过 `to.matched` 汇总父子级权限。

### TC-03：转包页点击“新建入库单”

步骤：

1. 进入 `/transfer`。
2. 点击顶部“新建入库单”按钮。

预期结果：

- 页面跳转到 `/inbound/order?create=1&transferStatus=转包`。
- 入库单创建弹窗自动打开。

修复点：

- `Transfer.vue` 新增 `openInboundCreate()`。
- `Order.vue` 识别 `create=1`。

### TC-04：转包完成后查看目标入库单

步骤：

1. 在转包页选择已入库且未封存的源看板。
2. 输入转包数量。
3. 执行转包。
4. 在成功弹窗中点击“查看目标入库单”。

预期结果：

- 成功弹窗展示目标入库单号。
- 点击按钮后跳转到 `/inbound/order?docNo=目标入库单号`。
- 入库单列表按该单号查询。

修复点：

- 后端返回 `targetInboundDocNo`。
- 前端结果弹窗显示并跳转。

## 风险与后续验证

本次已完成构建和后端测试。建议在真实浏览器中再执行一次完整业务流：

1. admin 登录。
2. 进入用户权限管理。
3. 进入转包管理。
4. 点击“新建入库单”。
5. 执行一次拆包转包并点击“查看目标入库单”。

如果真实环境中仍提示无权限，优先检查浏览器是否保留了旧登录缓存，退出重新登录即可刷新 `wms_user`。
