# admin 权限与转包入库单按钮修复 - 代码核心逻辑解析

## 1. 路由权限修复

位置：`frontend/src/router/index.js`

```js
import { getToken, getUser } from '@/utils/auth'
```

路由守卫不再直接读取 `localStorage.wms_user_info`，而是复用认证工具：

```js
const userInfo = getUser() || {}
const userRole = userInfo.role
const requiredRoles = to.matched.flatMap((record) => record.meta.roles || [])
```

核心点：

- `getUser()` 和登录写入使用同一个 key：`wms_user`。
- `to.matched` 能拿到父级和子级路由记录。
- `flatMap` 汇总所有层级声明的角色要求。

判断逻辑：

```js
if (requiredRoles.length > 0 && !requiredRoles.includes(userRole)) {
  ElMessage.warning('无权限访问该页面')
  next('/dashboard')
} else {
  next()
}
```

这样 admin 不会因为读取错 key 被误判成无权限，嵌套路由也不会漏掉父级权限规则。

## 2. 转包结果返回目标入库单号

位置：`backend/src/main/java/com/example/wms/dto/transfer/TransferResultDTO.java`

```java
private String targetInboundDocNo;
```

这个字段用于把后端拆包时自动创建的入库单号带回前端。

位置：`backend/src/main/java/com/example/wms/service/impl/TransferServiceImpl.java`

```java
return new TransferResultDTO(
        source.getKanbanNo(),
        sourceQtyAfter,
        targetKanbanNo,
        isMergeMode ? targetLabel.getLabelQty() : transferQty,
        source.getMaterialCode(),
        source.getMaterialName(),
        source.getSupplierName(),
        stockArea,
        now,
        isMergeMode ? "合包" : "拆包",
        inboundDocNo
);
```

其中 `inboundDocNo` 的来源：

- 合包：复用目标看板已有入库单号。
- 拆包：生成新的入库单号，并创建目标入库单。

## 3. 转包页新增“新建入库单”

位置：`frontend/src/views/transfer/Transfer.vue`

```vue
<el-button @click="openInboundCreate">
  <el-icon><Plus /></el-icon>新建入库单
</el-button>
```

对应方法：

```js
function openInboundCreate() {
  router.push({
    path: '/inbound/order',
    query: { create: '1', transferStatus: '转包' }
  })
}
```

这个按钮不是只切换当前 Tab，而是进入入库单管理页，并要求目标页面打开创建弹窗。

## 4. 转包成功后查看目标入库单

位置：`frontend/src/views/transfer/Transfer.vue`

```vue
<el-descriptions-item v-if="transferResult.targetInboundDocNo" label="目标入库单号">
  {{ transferResult.targetInboundDocNo }}
</el-descriptions-item>
```

弹窗底部按钮：

```vue
<el-button v-if="transferResult.targetInboundDocNo" @click="goTargetInboundOrder">
  查看目标入库单
</el-button>
```

跳转逻辑：

```js
function goTargetInboundOrder() {
  const docNo = transferResult.value.targetInboundDocNo
  resultVisible.value = false
  router.push({
    path: '/inbound/order',
    query: docNo ? { docNo } : undefined
  })
}
```

这样用户完成转包后，可以直接查看系统自动生成或复用的目标入库单。

## 5. 入库单页识别创建参数

位置：`frontend/src/views/inbound/Order.vue`

```js
onMounted(() => {
  openCreateDialogFromQuery()
  fetchOrders()
})
```

监听路由参数：

```js
watch(
  () => route.query.create,
  () => {
    openCreateDialogFromQuery()
  }
)
```

打开弹窗：

```js
function openCreateDialogFromQuery() {
  if (route.query.create === '1') {
    createVisible.value = true
  }
}
```

这样从转包页跳转到 `/inbound/order?create=1` 时，用户能直接看到创建入库单弹窗，而不是只停留在列表页。
