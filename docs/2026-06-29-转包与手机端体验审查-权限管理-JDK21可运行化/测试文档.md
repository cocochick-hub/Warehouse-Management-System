# 测试文档：转包与手机端体验审查、权限管理、JDK21 可运行化

## 一、测试环境

- 操作系统：Windows 11
- 后端构建工具：Maven 3.9.10
- 后端推荐 JDK：Microsoft JDK 21.0.8
- 当前系统默认 Java：OpenJDK 24
- PC 前端：Vite + Vue 3 + Element Plus
- 手机端：uni-app + Vue 3

## 二、后端测试

### 测试命令 1：系统默认 Java 24

```powershell
cd backend
mvn test
```

预期结果：

- 如果系统默认 Java 可兼容，应完成编译和测试。

实际结果：

- 失败。
- 错误核心信息：

```text
Fatal error compiling: java.lang.ExceptionInInitializerError: com.sun.tools.javac.code.TypeTag :: UNKNOWN
```

结果解析：

- 这是 Lombok 与 JDK 24 javac 内部 API 的兼容问题。
- 不是转包代码或权限代码的业务编译错误。
- 因此本项目当前应固定使用 JDK 21。

### 测试命令 2：使用 JDK 21

```powershell
cd backend
$env:JAVA_HOME='C:\Users\86139\.jdks\ms-21.0.8'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn test
```

预期结果：

- 后端编译通过。
- 单元测试和集成测试通过。

实际结果：

```text
Tests run: 76, Failures: 0, Errors: 0, Skipped: 2
BUILD SUCCESS
```

结果解析：

- JDK 21 下后端完整测试通过。
- 转包模块测试共 29 条通过，说明拆包、合包、异常校验、历史记录等逻辑可运行。

### 测试命令 3：JDK21 包装脚本

```powershell
cd backend
.\mvn-jdk21.ps1 -version
```

预期结果：

- Maven 显示 Java version 为 21.0.8。

实际结果：

```text
Java version: 21.0.8, vendor: Microsoft, runtime: C:\Users\86139\.jdks\ms-21.0.8
```

结果解析：

- 脚本能正确切换到 JDK 21。
- 后续推荐使用：

```powershell
.\mvn-jdk21.ps1 test
.\mvn-jdk21.ps1 spring-boot:run
```

## 五、功能测试建议

### PC 转包管理

建议手工验证：

1. 登录管理员或操作员账号。
2. 进入“转包管理”。
3. 点击右上角“新建转包”。
4. 输入一个已入库、未封存的源看板号，点击“选择源看板”。
5. 输入转包数量。
6. 不填目标看板号，执行拆包。
7. 填写已存在同物料目标看板号，执行合包。
8. 到“转包历史”Tab 检查记录。

### 权限管理

建议手工验证：

1. 使用 `admin` 登录。
2. 侧边栏应看到“用户权限”。
3. 新增一个 `operator` 用户。
4. 修改该用户为 `manager`。
5. 禁用该用户。
6. 用被禁用用户登录，应被拒绝。
7. 使用非 admin 用户访问 `/system/users`，前端应拦截，后端 `/api/admin/users` 也应返回 403。

## 六、当前遗留风险

1. 系统默认 Java 24 不能直接跑后端，需要使用 JDK 21 或升级 Lombok/Spring Boot 生态后再评估。
2. PC 前端构建依赖 `frontend/node_modules`，新环境需要先执行 `npm install`。
3. 手机端转包没有真机扫码验证，本次只验证了 H5 构建。
4. 权限管理目前是角色级权限，还不是细粒度按钮级权限，后续可扩展为权限点表。
