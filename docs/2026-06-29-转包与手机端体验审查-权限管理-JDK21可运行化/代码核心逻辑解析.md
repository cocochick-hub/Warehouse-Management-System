# 代码核心逻辑解析：转包、权限管理、JDK21

## 一、PC 转包入口优化

文件：`frontend/src/views/transfer/Transfer.vue`

新增页面右上角入口：

```vue
<template #actions>
  <el-button type="primary" @click="openCreateTransfer">
    <el-icon><Plus /></el-icon>新建转包
  </el-button>
</template>
```

核心含义：

- `PageContainer` 支持 `actions` 插槽。
- 用户进入页面后可以直接看到“新建转包”，不再依赖“点击表格行”这个隐式入口。

新增源看板号快速选择：

```vue
<el-form inline class="quick-create-form">
  <el-form-item label="源看板号">
    <el-input
      v-model="quickKanbanNo"
      placeholder="扫码或输入源看板号"
      clearable
      @keyup.enter="handleQuickSelect"
    />
  </el-form-item>
  <el-form-item>
    <el-button type="primary" :loading="quickLoading" @click="handleQuickSelect">
      <el-icon><Aim /></el-icon>选择源看板
    </el-button>
  </el-form-item>
</el-form>
```

核心逻辑：

```js
async function handleQuickSelect() {
  const kanbanNo = quickKanbanNo.value.trim()
  if (!kanbanNo) {
    ElMessage.warning('请输入源看板号')
    return
  }

  quickLoading.value = true
  try {
    const { data } = await getTransferLabel(kanbanNo)
    handleKanbanRowClick({
      ...data,
      availableQty: data.availableQty ?? data.labelQty
    })
  } finally {
    quickLoading.value = false
  }
}
```

解析：

- 调用已有 `/api/transfer/label` 查询源看板。
- 查询成功后复用原来的 `handleKanbanRowClick` 校验逻辑。
- 如果接口没有返回 `availableQty`，则用 `labelQty` 兜底。

## 二、后端转包核心逻辑

文件：`backend/src/main/java/com/example/wms/service/impl/TransferServiceImpl.java`

核心流程：

```java
InboundKanbanLabel source = kanbanLabelRepository.findByKanbanNoWithLock(request.getSourceKanbanNo())
        .orElseThrow(() -> new EntityNotFoundException("看板号不存在: " + request.getSourceKanbanNo()));
```

解析：

- 使用带锁查询源看板，避免并发转包导致超扣。

```java
int availableQty = calculateAvailableQty(source);

if (transferQty > availableQty) {
    throw new IllegalArgumentException(
            "转移数量(" + transferQty + ")超过源看板可用数量(" + availableQty + ")");
}
```

解析：

- 可用数量 = 当前看板数量 - 冻结数量。
- 转包数量不能大于可用数量。

```java
boolean userSpecified = targetKanbanNo != null && !targetKanbanNo.trim().isEmpty();
boolean isMergeMode = false;
InboundKanbanLabel targetLabel = null;

if (userSpecified) {
    targetLabel = kanbanLabelRepository.findByKanbanNo(targetKanbanNo).orElse(null);
    if (targetLabel != null) {
        isMergeMode = true;
    }
}
```

解析：

- 用户填写目标看板号且目标存在，就是合包。
- 没填或目标不存在，就是拆包。

```java
int updated = kanbanLabelRepository.decreaseLabelQty(source.getKanbanNo(), transferQty);
if (updated == 0) {
    throw new IllegalStateException("源看板数量已变化，请重试");
}
```

解析：

- 使用数据库原子扣减。
- 如果扣减失败，说明并发下源看板数量已变化，要求用户重试。

## 三、权限管理后端逻辑

文件：`backend/src/main/java/com/example/wms/controller/AdminUserController.java`

```java
@RestController
@RequestMapping("/api/admin/users")
public class AdminUserController {
    @GetMapping
    public ApiResult<List<ManagedUserDTO>> listUsers() {
        return ApiResult.success(userService.listManagedUsers());
    }
}
```

解析：

- 管理员用户管理统一放在 `/api/admin/users`。
- 返回统一 `ApiResult` 格式，兼容现有前端拦截器。

文件：`backend/src/main/java/com/example/wms/config/SecurityConfig.java`

```java
.antMatchers("/api/admin/**").hasRole("admin")
```

解析：

- JWT 过滤器会把角色转成 `ROLE_admin`、`ROLE_manager`、`ROLE_operator`。
- `hasRole("admin")` 会匹配 `ROLE_admin`。
- 因此非管理员即使绕过前端菜单，也会被后端拒绝。

文件：`backend/src/main/java/com/example/wms/service/impl/UserServiceImpl.java`

```java
private static final Set<String> ALLOWED_ROLES = Set.of("admin", "manager", "operator");
```

解析：

- 角色白名单防止写入未知角色。

```java
user.setPassword(passwordEncoder.encode(request.getPassword()));
```

解析：

- 管理员创建用户时，密码仍然走 BCrypt 加密，不保存明文。

## 四、权限管理前端逻辑

文件：`frontend/src/views/system/UserPermission.vue`

核心能力：

```vue
<el-select
  v-model="row.role"
  size="small"
  :disabled="row.username === currentUsername"
  @change="(role) => handleRoleChange(row, role)"
>
```

解析：

- 管理员可以直接在表格中修改别人的角色。
- 当前登录用户不能在页面上修改自己的角色，避免误操作把自己降权。

```vue
<el-switch
  v-model="row.status"
  :active-value="1"
  :inactive-value="0"
  :disabled="row.username === currentUsername"
  @change="(status) => handleStatusChange(row, status)"
/>
```

解析：

- 管理员可以启用或禁用账号。
- 禁用后该用户无法登录。

## 五、JDK21 运行脚本

文件：`backend/mvn-jdk21.ps1`

```powershell
$jdkHome = Join-Path $env:USERPROFILE ".jdks\ms-21.0.8"
$env:JAVA_HOME = $jdkHome
$env:Path = "$env:JAVA_HOME\bin;$env:Path"

& mvn @MavenArgs
exit $LASTEXITCODE
```

解析：

- 不依赖系统默认 `java`。
- 每次执行 Maven 前强制使用本机 JDK 21。
- 可以传递任意 Maven 参数，例如 `test`、`spring-boot:run`。
