# 入库模块演示流程文档

## 1. 文档说明

- 用途：用于课堂演示、小组汇报、验收展示
- 目标时长：3 ~ 5 分钟
- 最后更新：2026-06-08

## 2. 演示前准备

### 2.1 环境确认

- MySQL80 服务正常运行
- `wms_db` 已初始化，且入库模块演示数据可用
- 后端已启动：`http://localhost:8080`
- 前端已启动并可访问系统页面
- 测试账号可登录：`admin / admin123`

### 2.2 演示重点

本轮不要试图讲完整 WMS，只聚焦以下闭环：

1. 创建入库单
2. 查看入库单详情
3. 进入手工入库
4. 录入实际入库数量
5. 展示状态自动流转
6. 展示库存联动结果

## 3. 标准演示流程

### 第一步：登录系统

- 操作：使用 `admin / admin123` 登录
- 预期结果：进入系统首页，无报错
- 讲解口径：当前演示账号为管理员，后续操作基于统一权限入口

### 第二步：进入“入库单管理”页面

- 操作：从菜单进入入库模块的入库单管理页面
- 预期结果：能看到查询区、列表区、创建按钮
- 讲解口径：该页面负责入库单查询、创建和详情查看

### 第三步：创建一张新的入库单

- 操作：
  - 点击“新建入库单”
  - 选择/填写供应商
  - 新增 2 条明细
  - 填入物料号、物料名称、计划数量等信息
  - 提交
- 预期结果：
  - 创建成功提示出现
  - 列表自动回到第一页
  - 新建单据出现在列表顶部
  - 状态为 `未入库`
- 讲解口径：
  - 创建后系统会生成入库单号
  - 单据先记录计划数量，实际数量初始为 0

### 第四步：查看入库单详情

- 操作：点击刚创建单据的“详情”
- 预期结果：
  - 能看到单头信息
  - 能看到明细行
  - 若尚未入库，库存快照可能为空
- 讲解口径：
  - 详情页用于确认单据结构、计划数量和当前执行状态

### 第五步：跳转到“手工入库”页面

- 操作：点击列表中的“入库”按钮，或从菜单进入手工入库页
- 预期结果：
  - 打开待入库单据列表
  - 可以按单号、供应商、状态筛选
  - 目标单据可直接打开入库弹窗
- 讲解口径：
  - 手工入库页负责录入本次实际入库数量

### 第六步：先演示“部分完成”

- 操作：
  - 在入库弹窗中，只给其中一条明细录入部分数量
  - 提交入库
- 预期结果：
  - 提交成功提示出现
  - 单据状态变为 `部分完成`
  - 明细的待入库数量同步减少
- 讲解口径：
  - 系统根据计划总量和累计实收量自动判断状态，不需要手工改状态

### 第七步：再演示“已完成”

- 操作：
  - 再次打开同一单据
  - 补齐剩余待入库数量
  - 提交
- 预期结果：
  - 单据状态变为 `已完成`
  - 已入库总数等于计划总数
  - 已完成单据的“开始入库”按钮不可继续使用
- 讲解口径：
  - 这一步说明系统闭环已经跑通：计划数量、实际数量和状态流转是一致的

### 第八步：展示库存联动结果

- 操作：
  - 查看详情弹窗或入库弹窗中的库存快照表
- 预期结果：
  - 能看到物料对应的 `当前库存`
  - 能看到 `最近入库单号`
  - 能看到 `最近入库时间`
- 讲解口径：
  - 本轮已实现入库后最小库存联动，能展示库存数量变化结果

## 4. 演示中的注意事项

### 建议避免的操作

- 不要现场重建数据库
- 不要临时修改 `application.yml`
- 不要讲太多“以后要做什么”，优先展示已经跑通的能力
- 不要把范围拉到出库、批次、库位等未完成模块

### 推荐强调的亮点

1. 前后端已打通完整闭环
2. 状态流转自动计算
3. 超收和空提交有校验
4. 入库完成后能看到库存联动结果

## 5. 常见问答口径

### 如果被问到“为什么没有批次和库位”

推荐回答：

> 这一轮按课程阶段目标，先完成入库模块最小闭环。批次、库位和库存流水属于下一阶段库存精细化管理范围，已经在遗留问题文档中单独登记。

### 如果被问到“为什么不能编辑入库单”

推荐回答：

> 本轮重点是创建、执行入库和状态联动，因此优先保证闭环跑通。编辑接口已经在设计文档里预留，但本轮没有纳入实现范围。

### 如果被问到“单号生成是否能支持正式生产”

推荐回答：

> 当前方案足够支撑单机演示和课程验收，但如果未来走多实例正式部署，需要升级成更严格的唯一单号方案。

2. 演示过程中能清楚说明状态流转和库存联动
3. 对已知遗留问题有统一口径，不会临场失焦
